控制器方法
@RequestMapping(
    value = "/employee/{id}",
    method = RequestMethod.DELETE
)
public String deleteEmployee(@PathVariable int id){
    empDao.delete(id);
	//删除操作，参数id执行完操作后就不再需要了，使用重定向至/employee请求
    return "redirect:/employee";
}
以上内容完成之后，由于在script标签 th:src="@{/static/js/vue.js}" 属性中，对vue.js静态资源的访问会被前端控制器捕获(非.jsp结尾)，所以要额外在springMVC.xml配置文件中开启默认控制器defaultServlet
<mvc:default-servlet-handler/>
<!--开启对静态资源的访问，必须同时开启annotation driven注解-->
<mvc:annotation-driven/>
默认控制器是如何发挥作用的

当我们通过@{/static/js/vue.js}访问静态资源时，请求会先被前端控制器处理，当前端控制器找不到对应的资源时，会交给默认控制器defaultServlet来处理，如果默认控制器还是找不到资源，404(此404不同于前端控制器的404，不是由前端控制器抛出的，是由默认控制器抛出的)

@RequestMapping(
    value = "/employee/{id}",
    method = RequestMethod.GET
)
public String toUpdate(@PathVariable int id, Model model){
	model.addAttribute("employee", empDao.get(id));
	return "updateEmployee";
    //根据id获取对应员工对象，并共享到域对象中，转发到修改页面 updateEmployee.html
}

<!--updateEmployee.html 回显数据并修改提交-->
<form th:action="@{/employee}" method="post">
    <input type="hidden" name="_method" value="PUT">
    id:<input type="number" name="id" th:value=${employee.id} readonly><br>
    <!--设置id属性回显并只读-->
    lastName:<input type="text" name="lastName" th:value="${employee.lastName}"><br>
    email:<input type="text" name="email" th:value="${employee.email}"><br>
    gender:<input type=radio name="gender" value="1" th:field="${employee.gender}">男
    <input type="radio" name="gender" value="0" th:field="${employee.gender}">女
    <!--使用th:field="${employee.gender}"，如果该值和value一致，会自动添加 checked="checked" 属性-->
    <input type="submit" value="提交修改">
</form>

@RequestMapping(
    value = "/employee",
    method = RequestMethod.PUT
)
public String executeUpdate(Employee employee){
	empDao.save(employee);
	return "redirect:/employee";//重定向至查询页面
}
