单例模式

Java中，可用static来实现单例模式

饿汉式

(优点：天然线程安全 ； 缺点：对象存续时间过长，当短时间内不需要此对象时会浪费空间)

class New1 {
	private New1() {
		//构造方法要私有化，不允许外部直接调用构造方法
	}
   
	private static New1 new1 = new New1();

	public static New1 getInstance() {
		return new1;//不管用不用直接创建，饥不择食
	}

}

懒汉式

(优点：延迟对象的创建，不会浪费空间 ； 缺点：需要手动加锁实现线程安全)

class New2 {
	private New2() {
		//构造方法要私有化，不允许外部直接调用构造方法
	}

	private static New2 instance = null;

	public static New2 getInstance() {
		if (instance == null)
			instance = new New2();
		return instance;//什么时候用什么时候创建，拖延症
	}
}

不论调用多少次getInstance(),返回的都是同一对象

应用场景：

1.网站计数器，一般使用单例模式实现，否则难以同步 2.应用程序的日志应用，一般是由于共享的日志文件一直处于打开状态，因为只能有一个实例去操作，否则内容不好追加 3.数据库连接池 4.读取配置文件的类，一般也只有一个对象

模板方法(多态的应用)

Java中抽象类体现的就是一种模板模式的设计，抽象类作为多个子类的通用模板，子类在抽象类的基础上进行扩、改造，但子类 总体上会保留抽象类的行为方式。

解决的问题

当功能内部一部分实现是确定的，一部分实现是不确定的，这是可以将不确定的部分暴露出去，让子类实现

代理模式

public class StarTest {
	public static void main(String[] args) {
		RealStar rs = new RealStar();
		ProxyStar ps = new ProxyStar(rs);
		System.out.println();
	}
}

interface Star {
	void confer();// 面谈
	void signContract();// 签合同
	void bookTicket();// 订票
	void sing();// 演唱
	void collectMoney();// 收钱
}

class RealStar implements Star {

	@Override
	public void confer() {
		// TODO Auto-generated method stub

	}
	@Override
	public void signContract() {
		// TODO Auto-generated method stub
		System.out.println("The real star signs contract");
	}
	@Override
	public void bookTicket() {
		// TODO Auto-generated method stub

	}
	@Override
	public void sing() {
		// TODO Auto-generated method stub
		System.out.println("The real star sings");
	}
	@Override
	public void collectMoney() {
		// TODO Auto-generated method stub
		System.out.println("The real star collects money");
	}
}

class ProxyStar implements Star {
	private Star star;

	public ProxyStar(Star star) {
		// TODO Auto-generated constructor stub
		this.star = star;
	}
	@Override
	public void confer() {
		// TODO Auto-generated method stub
		System.out.println("Proxy confer");
	}
	@Override
	public void signContract() {
		// TODO Auto-generated method stub
		System.out.println("Proxy consider");
		System.out.println("Proxy thinks its good");
		star.signContract();
	}
	@Override
	public void bookTicket() {
		// TODO Auto-generated method stub
		System.out.println("Proxy books ticket");
	}
	@Override
	public void sing() {
		// TODO Auto-generated method stub
		star.sing();
	}
	@Override
	public void collectMoney() {
		// TODO Auto-generated method stub
		star.collectMoney();
	}
}

输出结果：

Proxy confer
Proxy consider
Proxy thinks its good
The real star signs contract
Proxy books ticket
The real star sings
The real star collects money

应用场景：

安全代理：屏蔽对真实角色的直接访问
远程代理：通过代理类处理远程方法调用(RMI)
延迟加载：先加载轻量级的代理对象，真正需要再加载真实对象

分类

  • 动态代理 (动态生成代理类)
