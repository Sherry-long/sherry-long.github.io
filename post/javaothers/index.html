 Java对象创建在堆上，对象引用在堆栈上
 对象为什么不在堆上：创建程序时，Java系统必须知道存储在堆栈内的所有项的确切生命周期，以便上下移动堆栈指针（分配新内存以及回收内存）
                     在堆中存储时，编译器不需要知道存储的数据在堆中存活多久，更有灵活性，但是在堆中进行内存分配和清理比堆栈花时间更多
 对于基本类型，将他们存储在堆栈上不是很高效，new时，创建一个并非是引用的“自动“变量，这个变量直接存储值，并存储在堆栈上，更加高效。

== 与 .equals() 方法的区别

==：用于比较基本数据类型时，比较的是变量保存的内容，不一定要类型相同，值相同即为true

    用于比较引用数据类型时，比较的是两个对象的地址，即两个引用是否指向内存中的同一个地址
	
.equals()：Object类中的 .equals() 方法就是通过 == 来实现的。

           只能适用于引用数据类型(继承于Object)
		   
           像String, Date, File, 包装类等都重写了 .equals() 方法，重写之后，比较的是两个对象的内容是否相同

基本类型、包装类

Java提供了8种基本数据类型对应的包装类，使得基本数据类型的变量具有类的特征

int — Integer

float — Float

double — Double

short — Short

long — Long

boolean — Boolean

char — Character

byte — Byte

JDK 5.0以后，增加了自动装箱，自动拆箱机制。

在使用包装类进行符号运算时，以及在向以包装类为形参的方法中传入基本数据类型时，会触发自动拆箱(前者)，自动装箱机制(后者)。

int num2 = 10;
Integer num1 = num2; //无需使用构造函数,自动装箱


//Integer内部定义了IntegerCache结构，IntegerCache中定义了一个Integer[],包含了-128~127的整数。当我们使用自动装箱机制给Integer赋值
//范围在-128~127之内时，可以直接使用数组中的元素，无需new。当超出这一范围时，Integer会自动帮我们new一个Integer。
Integer num3 = 1;
Integer num4 = 1;
System.out.println(num3 == num4); // 自动装箱赋值在范围内，num3和num4是两个数，true

Integer num5 = 128;
Integer num6 = 128;
System.out.println(num5 == num6);// 自动装箱赋值在范围外，此时Integer其实是new了两个Integer，所以此时比较的是两个对象，false

三目运算符相关

? : 三目运算符，冒号左右两边应当为同一类型。

Object o = true ? new Integer(1) : new Double(2.0)
syso(o);
//因为此时冒号两边类型不同，所以左边Integer将被转型成Double,所以此时输出为 1.0

代码块(多了一种赋值方式)

类的成员之一

作用：用来初始化类、对象

如果需要关键字修饰，只能使用static来修饰(静态代码块 与 非静态代码块)

静态代码块：

内部可以有输出语句，在类初次被加载的时候，会按照static块的顺序来执行每个static块，并且只会执行一次

只可以调用静态属性或方法，不可以调用非静态属性或方法

可写在类中除了方法内的任意位置，任何方法都不行！

作用：初始化类信息

非静态代码块：

内部可以有输出语句，在对象被创建时执行

可以调用静态属性方法与非静态属性方法

作用：可以在创建对象时，对对象的属性等进行初始化

对属性赋值的先后顺序

  1. 默认初始化
  2. 显式初始化 / 代码块赋值
  3. 构造器初始化
  4. 对象.属性 或 对象.方法
