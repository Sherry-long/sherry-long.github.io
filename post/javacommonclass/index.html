String相关

String

String是一个final类，无法被继承，支持序列化

String不可变字符序列，String内部并没有修改内容相应方法，每次修改String的方法其实质是创建一个新对象并将原引用指向它

String str1 = "abc"; // 字面量，区别于new
String str2 = "abc";
//此时str1和str2指向的是常量池中的同一地址
//字符串常量池中不会存储两个相同的字符串
str1 == str2.sout //结果为true

  • String不可变性体现：

      1. 当对字符串重新赋值时，需要重新指定内存区域赋值
  2. 当对现有字符串进行连接操作时( + )，也需要重新制定内存区域进行赋值
  3. 当调用String的 replace(oldChar, newChar) 方法修改指定字符或字符串时，也需要重新制定内存区域进行赋值

String对象的创建

String str = "hello";

String s1 = new String();// this.value = new char[0];

String s2 = new String(String original);// this.value = original.value;

String s3 = new String(char[] a);// this.value = Arrays.copyOf(value, value.length);

String s4 = new String(char[] a, int startIndex, int count);
String str = “abc”; 与 String str = new String(“abc”) 有什么区别
		String str1 = "abc"; //str指向常量池
        String str2 = "abc"; 
        String str3 = new String("abc"); //str3指向堆空间，堆空间对应的地址中存放的数据是常量池的地址
        String str4 = new String("abc");
        System.out.println(str1 == str2);//true
        System.out.println(str2 == str3);//false
        System.out.println(str3 == str4);//false

通过字面量的方式创建String，是将字面量在常量池中的地址赋予了引用 通过new的方式创建，是在堆空间中开辟空间以后对应的地址值

注意，new String(String original) 构造器已经不推荐使用
    Initializes a newly created String object so that it represents the same sequence of characters as the argument; 
    in other words, the newly created string is a copy of the argument string. 
    Unless an explicit copy of original is needed, use of this constructor is unnecessary since Strings are immutable.
	
    初始化新创建的字符串对象，使其表示与 参数 相同的字符序列；换句话说，新创建的字符串是 参数字符串 的一个副本
    由于String是不可变的，所以此构造器不必使用，除非你明确需要一个 参数字符串 的副本

String常量 + ：String str1 = “123” + “456”; === String str1 = “123456”;

String变量 + ：String str1 = “123”, str2 = “456”, str3 = str1 + str2; === 先把str1转成SBuilder，使用append(str2)，再toString()转回来

  • 常量与常量的拼接结果在常量池。且常量池中不会存在相同内容的常量

  • 只要其中有一个是变量，结果就在堆中

  • 如果拼接的结果调用intern()方法，返回值就在常量池中

StringBuffer, StringBuilder

  • StringBuffer 代表可变的字符序列，可以对字符串内容进行增删，不会产生新的对象

  • 很多方法与String相同

  • 作为参数传递时，方法内部可以改变其值

StringBuffer常用方法

StringBuffer append(xxx) //多个重载方法，用于进行字符串拼接
StringBuffer delete(int start, int end) //删除指定位置内容
StringBuffer replace(int start, int end, String str) //把 [start, end) 左闭右开 位置替换为str
StringBuffer insert(int offset, xxx) //在指定位置插入xxx
StringBuffer reverse() //字符序列逆转
以上支持方法链，且无论接不接受返回值，修改都能成功

String, StringBuffer, StringBuilder 之间的异同

String –是不可变字符序列，
StringBuffer –可变字符序列，线程安全，效率低（多线程，且存在临界资源时，选择StringBuffer）
StringBuilder –可变字符序列，线程不安全，效率高（非多线程，首选StringBuilder）
三者底层都是使用char[]数组存储

StringBuffer(StringBuilder)调用空参构造器，默认长度是16。调用有参构造器，初始化长度为“参数”.Length() + 16，但是在调用length()时，显示的仍然是正常的长度。

当进行扩容时，容量为原来的 2倍 + 2 ，同时将原有数组中的元素复制到新的数组中

SimpleDateFormat

Date类的API不易于国际化，大部分被废弃了，java.text.SimpleDateFormat类是一个不与语言环境有关的方式来格式化和解析日期的具体类

  • 格式化：日期 -> 文本

  • 解析：文本 -> 日期

//格式化：
SimpleDateFormat()//默认的模式和语言环境创建对象
public SimpleDateFormat(String pattern)//该构造方法可以用参数pattern指定的格式创建一个对象，该对象调用：
public String format(Date date)//方法格式化时间对象date

//解析：
public Date parse(String source)//从给定的字符串解析文本，生成一个日期

//自定义格式
SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd hh:mm:ss");
Date date = new Date();
String s = sdf.format(date);
System.out.println(date);
System.out.println(s);
System.out.println(sdf.parse(s));

//输出：
Sun Sep 04 11:40:16 CST 2022
2022-09-04 11:40:16
Sun Sep 04 11:40:16 CST 2022

LocalDate, LocalTime, LocalDateTime

@Test
    public void test3(){
        LocalDate localDate = LocalDate.now();//获取当前日期
        LocalTime localTime = LocalTime.now();//获取当前时间
        LocalDateTime localDateTime = LocalDateTime.now();//获取当前日期 + 时间

        System.out.println(localDate);//2022-09-04
        System.out.println(localTime);//17:25:33.613702500
        System.out.println(localDateTime);//2022-09-04T17:25:33.613702500

        //获取指定的日期时间，没有偏移量
        LocalDateTime localDateTime1 = LocalDateTime.of(2022, 9, 4, 17, 12, 56);
        System.out.println(localDateTime1);//2022-09-04T17:12:56
		
		//获取方法
        System.out.println(localDateTime.getDayOfMonth());//4
        System.out.println(localDateTime.getDayOfWeek());//SUNDAY
        System.out.println(localDateTime.getDayOfYear());//247
        System.out.println(localDateTime.getMonthValue());//9
		
		//设置属性方法，不可变性
        LocalDate localDate1 = localDate.withDayOfMonth(24);
        System.out.println(localDate1);//2022-09-24
        System.out.println(localDate);//2022-09-04
		
		//增加，不可变性
		System.out.println(localDate.plusMonths(2));//2022-11-04
        System.out.println(localDate);//2022-09-04
    }

Instant

时间线上的一个瞬时点。可能被用来记录APP中的事件时间戳。Instant精度很高。

@Test
    public void test4(){
        //获取本初子午线对应的时间
        Instant instant = Instant.now();
        System.out.println(instant);
        //添加时间偏移量
        OffsetDateTime offsetDateTime = instant.atOffset(ZoneOffset.ofHours(8));
        System.out.println(offsetDateTime);
        //获取从1970-1-1 00：00：00开始到现在的毫秒数
        System.out.println(instant.toEpochMilli());
    }
