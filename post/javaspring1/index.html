IoC操作 Bean管理(基于xml配置文件)

  • 什么是Bean管理： Bean管理指的是两个操作：1，创建对象 2，注入属性

基于xml方式创建对象

<bean id="user" class="com.sherry.Spring.User"></bean>

在spring配置文件中，使用bean标签，标签里面添加对应属性，就可以实现对象创建

在bean标签中有很多属性，常用属性：
id：唯一标识
class：类全路径（包类路径）

创建对象的时候，默认执行无参构造方法，实体类内部要么无构造方法，要么显式定义无参构造方法，否则会报错

基于xml方式注入属性

第一种注入方式：在实体类内部编写set方法，通过方法进行注入

（可以使用p名称空间进行简化）

//实体类
public class User {

    private String userName;
    private int userAge;

    public void setUserName(String userName) {
        this.userName = userName;
    }

    public void setUserAge(int userAge) {
        this.userAge = userAge;
    }

    public void add(){
        System.out.println("add .................");

    }
    public void show(){
        System.out.println(userName + ": " + userAge);
    }
}

//配置文件
<beans ...>

    <bean id="user" class="com.sherry.Spring.User">
        <property name="userName" value="Ming"></property>
        <property name="userAge" value="23"></property>
    </bean>
	
</beans>

第二种注入方式：使用有参构造进行注入
//实体类
public class User {

    private String userName;
    private int userAge;

//在实体类中定义有参构造器

        this.userName = userName;
        this.userAge = userAge;
    }

<beans ...>

    <bean id="user" class="com.sherry.Spring.User">

工厂bean

Spring有两种类型的bean，一种是普通bean（上面都是），一种是工厂bean（FactoryBean）

  • 普通bean：在配置文件中配置的类型就是返回的bean的类型

  • 工厂bean：在配置文件中定义的bean类型可以和返回类型不一样

第一步：创建类，作为工厂bean，实现FactoryBean接口；第二步：实现接口中的抽象方法，在方法中定义返回的bean类型

bean作用域

在Spring里面，可以设置创建bean实例是单实例还是多实例

默认情况下，bean是单实例对象

  • 如何设置
//在配置文件的bean标签中，有一个属性scope，此属性有两个候选值，singleton(单例)，prototype(多例)
//还有request, session，此处略
<bean id="***" class="***" scope="prototype">
	<...略
</bean>

//singleton与prototype区别，除了设置单多例之外
//singleton 加载Spring配置文件时便会创建单实例对象
//prototype 不在加载时创建，在调用getBean时创建对象

bean生命周期

1）通过构造器创建bean实例（无参构造）

2）为bean的属性设置值、对其他bean的引用

3）调用bean的初始化方法（需要进行配置初始化方法，在bean标签中设置属性initMethod）

4）获取使用bean

5）当容器关闭时，调用bean的销毁方法（需要进行配置销毁的方法，在bean标签中设置属性destroyMethod）

  • 在加入bean的后置处理器之后，生命周期变为7部分

分别在3）之前和之后，把bean实例传递给bean后置处理器的方法

实现演示
//创建实体类，并实现BeanPostProcessor接口（bean后置处理器接口）
public class BPPBean implements BeanPostProcessor {
    @Override
    public Object postProcessBeforeInitialization(Object bean, String beanName) throws BeansException {
        return BeanPostProcessor.super.postProcessBeforeInitialization(bean, beanName);
    }

    @Override
    public Object postProcessAfterInitialization(Object bean, String beanName) throws BeansException {
        return BeanPostProcessor.super.postProcessAfterInitialization(bean, beanName);
    }
}

//配置文件
<bean id="MyBPP" class="com.sherry.Spring.bean.BPPBean"></bean>

//在配置文件配置实现BPP接口的bean之后，对当前配置文件的所有bean都有效

自动装配

根据指定的装配规则，Spring自动将匹配的属性值注入

1）根据属性名称装配（无视大小写）

//配置文件
//***类中有一个名为address的属性
<bean id="***" class="***" autowire="byName"></bean>
//Spring会自动识别id与属性名，进行自动装配
//bean名必须与属性名一致（大小写可不一致）
<bean id="address" class="***.Address">略</bean>

2）根据属性类型装配

//配置文件
//***类中有一个类型为Address的属性
<bean id="***" class="***" autowire="byName"></bean>
//Spring会自动识别bean类型与属性类型，进行自动装配
//bean名不需要与属性名一致，但是同一类型不能有多个bean，否则无法装配
<bean id="not_address" class="***.Address">略</bean>
